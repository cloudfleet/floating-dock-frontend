Why?

When we created CloudFleet we noticed a gap in the infrastructure we needed.

Docker Hub does not support builds for different architectures. However most containers are architecture agnostic and can be built for any architecture.

With ARM servers on the horizon marina.io strives to reproduce the wonderful ecosystem of the Docker Hub container by container, to make it easy to build your containers for any architecture.

How?

Before building the container, we adapt the FROM line to use the proper armhf container as a parent.

  • Where possible we use the official armhf or aarch64 containers by Docker.
  • If the right one is not available we try to rebuild them from the library specs.
  • If this is also not possible we build an image that is as similar as possible. You can suggest more images or improvements to existing ones on Github.

Features

Automated Builds for armhf and aarch64

For now marina.io builds containers for the architectures armhf and aarch64. These are the architectures the Raspberry Pi operating systems are running on.
The images are build on our little Raspberry Pi farm.

Accepts Webhooks from GitHub

Simply setup a webhook on your GitHub repository or organization and atomatically trigger a build on each commit.

Replicates Structure of Docker Hub

Easily create automated builds the same way you do on Docker Hub.

Reuse Your Dockerfile

You don't have to change anything in your code.
Just reuse your Dockerfile and our build system takes care of the rest. If it doesn't work feel free to reach out to us so we can make it work.

Missing Something?
If you are missing a feature or find a bug, let us know on GitHub. Only together can we make DevOps on ARM easier!
